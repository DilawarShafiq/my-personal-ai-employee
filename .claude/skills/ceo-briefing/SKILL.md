# ceo-briefing — the headline feature

This is the feature judges will remember. It transforms the AI Employee from a chatbot into a chief-of-staff. Execute with care.

## When to use
- `/Needs_Action/BRIEFING_TRIGGER_*.md` exists (cron-scheduled).
- User asks for "Monday briefing", "CEO briefing", "weekly report".
- Any Monday morning the orchestrator cycle starts and no briefing for
  today already exists in `/Briefings/`.

## Input sources (read, in this order)
1. `Company_Handbook.md` — tone + PHI rules.
2. `Business_Goals.md` — KPI targets and `verified: ✅` metrics.
3. Odoo MCP → `odoo_financial_snapshot` — revenue, AR, expenses.
   If Odoo is offline, use the graceful-degrade demo snapshot it returns.
4. `Accounting/Current_Month.md` — secondary source / cross-check.
5. `Done/` folder (last 7 days) — tasks the AI Employee completed.
6. `Logs/*.jsonl` (last 7 days) — AI actions taken + approvals.
7. `Pending_Approval/` — outstanding items needing CEO attention.
8. Social MCP → `social_weekly_summary` — posts-per-channel rollup.

## Output — `Briefings/<today>_Monday_Briefing.md`

## Rules
- **Honesty over optics.** If Odoo returned `live: false`, say so in the
  `data_sources` frontmatter.
- **Every number traceable.** No metric in the briefing without a source
  citation (handbook/goals row, Odoo call, log entry).
- **US healthcare framing.** Sales pipeline section uses clinic /
  billing-company verbs, not generic SaaS ("providers onboarded",
  "denials managed", "days-in-AR improved").
- **3 proactive suggestions, no more.** The briefing is for a CEO who
  has 4 minutes on a Monday morning. Each suggestion ends with an
  explicit CTA (create `/Pending_Approval/<file>` if action is needed).

## Output contract
- Single file at `Briefings/<YYYY-MM-DD>_Monday_Briefing.md`.
- Update `Dashboard.md` *Recent activity*: "CEO briefing generated: <path>".
- Write audit line.
- End turn with `<promise>CEO_BRIEFING_READY</promise>`.
